* 뉴스 로직 
Alpha Vantage News & Sentiment
가장 먼저 붙이기 좋습니다. tickers, topics, time_from/time_to, sort, limit이 있고 뉴스 감성 데이터까지 같이 줍니다. 현재 news_cache.json 구조에도 잘 맞습니다.
추천 용도: 종목별 뉴스 + 감성 라벨 + 구현 단순성.
출처: Alpha Vantage NEWS_SENTIMENT docs

Benzinga News API
비용을 감수할 수 있으면 가장 “시장 반응형 뉴스”에 가깝습니다. tickers, dateFrom/dateTo, publishedSince, importance, channels 같은 필터가 있고, 실시간/증분 수집에 적합합니다.
추천 용도: 미국 주식 중심, 시세에 영향 줄 만한 뉴스, 운영 품질.
출처: Benzinga News API docs

Financial Modeling Prep Stock News
구현이 단순하고 주식 뉴스 피드 목적에 잘 맞습니다. 최신 주식 뉴스에서 headline, snippet, URL, ticker를 주는 형태라 현재 UI에 넣기 쉽습니다.
추천 용도: 가볍게 Yahoo보다 넓은 주식 뉴스 피드 추가.
출처: FMP Stock News API

NewsAPI
범용 해외 뉴스 검색에는 좋습니다. /v2/everything에서 키워드, 도메인, 언어, 날짜, 정렬 조건을 줄 수 있습니다. 다만 “AAPL” 같은 티커 매칭보다 “Apple” 같은 키워드 검색에 가깝기 때문에 종목 매칭 오탐을 직접 줄여야 합니다.
추천 용도: CNBC, Reuters, Bloomberg류 도메인 필터 기반 보조 뉴스.
출처: NewsAPI Everything endpoint

GDELT
무료/글로벌/다국어 커버리지가 장점입니다. 65개 언어를 영어 검색어로 검색할 수 있고 JSON도 지원합니다. 대신 금융 종목 뉴스로 정제된 API는 아니라 노이즈가 많습니다.
추천 용도: 글로벌 매크로, 국가별 분위기, 테마/섹터 뉴스 모니터링.
출처: GDELT DOC 2.0 API

내 추천은 1차로 Alpha Vantage, 나중에 품질이 부족하면 Benzinga를 유료 옵션으로 붙이는 구조입니다. 구현은 provider 방식이 좋습니다.

news provider 우선순위
1. yfinance fallback
2. Alpha Vantage, API key 있으면 사용
3. FMP 또는 NewsAPI 보조
4. Benzinga, 유료 운영 전환 시 사용
중요한 점은 기사 전문을 저장하지 말고 ticker, title, publisher, summary/snippet, url, publishedAt, sentiment, source 정도만 캐시하는 게 안전합니다. UI도 지금 구조와 잘 맞습니다.
